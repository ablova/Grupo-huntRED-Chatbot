Flujos de Procesos

Flujo del Chatbot

🚀
Inicio
El candidato inicia la interacción, y el chatbot carga el contexto.
  • AuthenticationVerifica identidad con OAuth (`app/ats/services/auth.py`).
  • Context LoadRecupera datos de Redis y PostgreSQL (`app/ats/models/memory.py`).
  • Session StartInicia sesión WebSocket (`app/ats/services/websocket.py`).
🎯
Identificación
Analiza intenciones y perfil con NLP avanzado.
  • Intent ClassificationUsa spaCy en `app/ats/chatbot/nlp/nlp.py`.
  • Entity ExtractionIdentifica habilidades con modelos NER.
  • Sentiment AnalysisEvalúa tono con TensorFlow.
🏢
Unidad de Negocio
Selecciona unidad y activa flujo personalizado.
  • Unit DetectionCompara input en `app/ats/chatbot/workflow/common/common.py`.
  • Flow ActivationCarga flujo específico (e.g., huntRED, Amigro).
  • Context UpdateActualiza estado en `app/ats/chatbot/models/chat_state.py`.
📋
Flujo Específico
Ejecuta procesos como referencias o entrevistas.
  • Workflow ExecutionUsa Workflow Engine en `app/ats/chatbot/workflow`.
  • Integration CallsConsulta CRM (`app/ats/integrations/services/*.py`).
  • ValidationValida datos en `app/ats/utils/validators.py`.
🤖
Análisis ML
Aplica ML para scoring y recomendaciones.
  • ScoringCalcula puntajes en `app/ats/chatbot/core/gpt.py`.
  • RecommendationsUsa OpenAI embeddings.
  • PredictionsFit cultural con TensorFlow.
📊
Evaluación
Genera reportes y actualiza estado del candidato.
  • Report GenerationCrea PDF en `app/models.py`.
  • Status UpdatePersiste en PostgreSQL, notifica vía RabbitMQ.
  • FeedbackRecolecta datos para reentrenamiento.

ML-Driven Insights
Genera insights de compatibilidad.
